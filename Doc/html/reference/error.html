2.5.2 Members

⇧ 
type ErrorMessage

This union type represents an individual error message within the ErrorMessageList returned by a Parser function.

ErrorMessage values are always associated with a particular input stream position. For example, the error messages of the ErrorMessageList stored in the Reply<_,_>.Error field describe an error at the position of the Reply<_,_>.State value. The ParserError type is a convenient container type for binding together an ErrorMessageList with its associated position.

ErrorMessage has the following union cases:

⇧ 
| Expected of string

This ErrorMessage is reported when some expected input could not be parsed. The string argument describes the expected input.

This error message can be generated with the labeling operator <?>.

⇧ 
| Unexpected of string

This ErrorMessage is reported when some unexpected input was encountered. The string argument describes the unexpected input.

This error message is primarily generated by the notFollowedByL, notFollowedByChar and notFollowedByString parser primitives.

⇧ 
| Message of string

This ErrorMessage is reported if the error does not fit into other error categories. The string argument contains the error message.

This error message can be generated with the fail and failFatally primitives.

⇧ 
| CompoundError of string * Pos * ErrorMessageList

This ErrorMessage is reported if a “compound” failed to parse. The constructor arguments are a label for the compound parser and the error (Pos + ErrorMessageList) that caused the compound to fail to parse.

This error message is primarily generated by the compound‐labelling operator <??>.

⇧ 
| BacktrackPoint of Pos * ErrorMessageList

This ErrorMessage is reported if a parser backtracked after an error occurred. The constructor argument is the error (Pos + ErrorMessageList) that occurred before the backtracking.

This error message is primarily generated by the attempt, >>? and .>>? primitives.

⇧ 
| OtherError of obj

This ErrorMessage can be used to report application specific error data. The object argument contains the error data.

Important

The object argument is expected to be comparable by F#’s structural comparison function compare. OtherError instances with arguments that are not comparable are ignored by the ErrorMessageList methods.

To display OtherError values in error messages, you will have to define your own error printer, as the default printer ignores them.

⇧ 
type ErrorMessageList

This union type represents a list of error messages in which the order of the messages carries no meaning and any duplicates and empty messages are ignored. Essentially, an ErrorMessageList is constructed as a list, but interpreted as a set.

All ErrorMessage values in an ErrorMessageList are associated with the same input stream position. For example, the error messages stored in the Reply<_,_>.Error field describe an error at the position of the Reply<_,_>.State value. The ParserError type is a convenient container type for binding together an ErrorMessageList with its associated position.

The easiest way to process an ErrorMessageList as a set is to convert it into an actual set by using the ToSet member. For example, the ParserError.WriteTo method uses this method to filter out duplicates before printing out the messages.

In order to enforce set semantics in comparison operations, the ErrorMessageList overrides the Equals, GetHashCode and CompareTo methods.

⇧ 
| AddErrorMessage of ErrorMessage * ErrorMessageList

Creates a new ErrorMessageList with the given “head” and “tail”.

For example,

AddErrorMessage(Expected "a",
                (AddErrorMessage(Expected "b", NoErrorMessages))

creates an ErrorMessageList with two error messages.

⇧ 
| NoErrorMessages

The empty ErrorMessageList. This value is represented as null.

⇧ 
member ToSet: unit -> Set<ErrorMessage>

Converts the ErrorMessageList to a Set<ErrorMessageList>. Duplicate error messages and empty Expected, Unexpected and Message messages are filtered out when the list is converted to a set.

⇧ 
static member OfSeq: seq<ErrorMessage> -> ErrorMessageList

Converts a sequence of ErrorMessage values into an ErrorMessageList.

⇧

⇧ 
val unexpectedError: string -> ErrorMessageList

unexpectedError label creates an ErrorMessageList with a single Unexpected label message.

⇧ 
val messageError: string -> ErrorMessageList

messageError msg creates an ErrorMessageList with a single Message msg message.

⇧

otherError o creates an ErrorMessageList with a single OtherError o message.

⇧

⇧

⇧ 
val concatErrorMessages:
        ErrorMessageList -> ErrorMessageList -> ErrorMessageList

concatErrorMessages error1 error2 concatenates the two error message lists error1 and error2.

⇧ 
val mergeErrors:
        ErrorMessageList -> ErrorMessageList -> ErrorMessageList

mergeErrors error1 error2 is an inlined variant of concatErrorMessages error1 error2 that avoids the call to concatErrorMessages if error1 is empty (i.e. equals NoErrorMessages).

[<Sealed>]
type ParserError

ParserError is a simple container type for holding an ErrorMessageList together with its associated input stream position.

⇧ 
new: Pos * ErrorMessageList -> ParserError

Constructs a ParserError from an ErrorMessageList and its associated position.

⇧ 
member Pos: Pos

The input stream position of the parser error.

⇧ 
member ErrorMessageList: ErrorMessageList

The error messages of the parser error.

⇧ 
override ToString: unit -> string

Is equivalent to

use sw = new System.IO.StringWriter()
WriteTo(sw)
sw.ToString()
⇧ 
member ToString: streamWhereErrorOccurred: CharStream -> string

Is equivalent to

use sw = new System.IO.StringWriter()
WriteTo(sw, streamWhereErrorOccurred = streamWhereErrorOccurred)
sw.ToString()
⇧ 
member WriteTo:
         textWriter: System.IO.TextWriter
       * ?positionPrinter:
            (System.IO.TextWriter -> Pos -> string -> int -> unit)
       * ?streamWhereErrorOccurred: CharStream
       * ?columnWidth: int
       * ?initialIndention: string * ?indentionIncrement: string
       -> unit

Writes a string representation of the ParserError to the given TextWriter value.

The format of the position information can be customized by specifying the positionPrinter argument. The given function is expected to print a representation of the passed Pos value to the passed TextWriter value. If possible, it should indent text lines with the passed string and take into account the maximum column count (including indention) passed as the last argument.

If a value for the optional CharStream argument is given, the printed position information is augmented with the the line of text surrounding the error position, together with a ‘^’‐marker pointing to the exact location of the error in the input stream. The given CharStream must contain the original input for which the parser error was generated.

⇧ 
val printErrorLine:    CharStream -> index: int64 -> System.IO.TextWriter
                    -> indentionString: string -> columnWidth: int
                    -> unit

Prints the line of text surrounding the given index in the stream and marks the position of the index with a caret (^) on a second line. The output is indented with the given string and is restricted to the given column width (including the indention).
